## 一、项目目标

本设计以 ST89C52 单片机为核心，实现 **1Hz~20Hz 可调 SPWM 波形发生器**。系统包含以下功能：

- 输出 SPWM 波形
- 通过按键调节输出频率
- 通过两位共阳数码管显示当前频率

本项目当前的软件实现已经明确采用以下约束：

1. `Timer0` 用于 SPWM 输出
2. `Timer1` 仅用于产生系统节拍
3. **数码管刷新严格在主循环中完成，不通过定时器中断刷新**

---

## 二、系统原理

### 1. SPWM 生成原理

SPWM 通过改变 PWM 占空比，使输出波形在等效上逼近正弦波。

本项目采用查表法：

- 一个正弦周期离散为 100 个采样点
- 每个采样点对应一个 PWM 占空比
- `Timer0` 中断负责依次输出每个采样点对应的脉冲

频率范围为 `1Hz~20Hz`，通过修改采样周期实现频率调节。

### 2. 数码管显示原理

系统使用两位共阳数码管，段选由 `P1` 输出，位选由 `P2.0 / P2.1` 控制。

显示采用动态扫描，但刷新方式不是定时器中断，而是：

- 主循环持续调用 `display_task()`
- 每次只刷新一位数码管
- 十位和个位交替轮询

显示规则：

- `1~9Hz`：十位消隐，个位显示
- `10~20Hz`：两位正常显示

### 3. 按键处理原理

按键模块采用非阻塞状态机设计，支持：

- 消抖
- 长按
- 连发

`Timer1` 仅产生一个粗节拍信号，前台在收到节拍后执行按键扫描，不参与数码管刷新。

## 三、总体软件架构

当前代码架构如下：

```text
main.c                  程序入口
app.c / app.h           应用层调度
spwm.c / spwm.h         SPWM 输出模块
display.c / display.h   数码管显示模块
key.c / key.h           按键扫描模块
freq.c / freq.h         频率管理模块
tick.c / tick.h         系统节拍模块
config.h                板级配置
```

模块职责如下：

- `main`：只负责初始化和进入主循环
- `app`：负责前后台任务调度
- `display`：负责数码管前台刷新
- `key`：负责按键状态机扫描
- `freq`：负责频率状态管理
- `tick`：负责节拍标志传递
- `spwm`：负责波形输出

## 四、主循环与中断分工

### 1. 主循环职责

主循环中持续执行：

```c
while (1)
{
    app_task();
}
```

应用层任务中包含：

1. 调用 `display_task()` 刷新一位数码管
2. 检查系统节拍
3. 执行按键扫描
4. 根据频率值更新显示缓存

### 2. 中断职责

#### Timer0 中断

- 输出 SPWM 波形
- 不处理显示逻辑

#### Timer1 中断

- 只调用 `tick_notify()`
- 只用于产生按键扫描节拍
- **不进行数码管扫描**

这是一条必须保持的设计原则。

## 六、显示模块说明

### 1. 硬件连接

| 引脚 | 功能 | 宏定义 |
|------|------|--------|
| P1.0~P1.6 | 七段数码管 a~g 段 | `SEG_PORT` |
| P2.0 | 十位位选 | `DIG_TENS` |
| P2.1 | 个位位选 | `DIG_ONES` |

### 2. 软件实现特点

数码管模块位于 `display.c / display.h`，特点如下：

1. 数码管刷新不在定时器中断中执行
2. 前台每次只刷新一位
3. 切换顺序为：先关位选，再切段码，最后打开目标位
4. 小于 10 时十位自动消隐

这种设计更符合当前项目对显示时序可控性的要求。

---

## 七、SPWM 模块说明

SPWM 模块位于 `spwm.c / spwm.h`。

### 1. 算法方式

- 使用 `100` 点正弦查表
- 使用 `Timer0` 双相中断输出
- 频率相关参数采用预计算表降低 ISR 运算量

### 2. 关键参数

| 参数 | 数值 |
|------|------|
| 晶振频率 | 11.0592 MHz |
| 正弦采样点数 | 100 |
| 频率范围 | 1~20 Hz |
| PWM 分辨率 | 256 级 |
