## 一、设计背景与目的

本次课程设计以 ST89C52 单片机为核心，设计一款 1Hz~20Hz 频率可调的 SPWM 波形发生器。旨在将课堂所学的电力电子技术、单片机原理等知识应用于实践，掌握 SPWM 波形的生成原理、硬件电路设计与程序开发流程，提升电路调试与系统设计能力。

## 二、SPWM 原理概述

SPWM（正弦脉宽调制）技术通过对脉冲宽度进行正弦规律调制，使输出波形等效为正弦波。其核心原理是用高频载波（通常为三角波）与正弦调制波比较，根据比较结果控制开关器件的通断，生成等幅不等宽的脉冲序列。

本次设计中，调制波频率需在 1Hz~20Hz 范围内可调。通过单片机定时器中断，按正弦查找表输出不同占空比的 PWM 信号，实现正弦规律调制。具体方法如下：

1. **查表法**：预先将一个完整正弦周期离散为 100 个采样点，每个采样点对应一个 PWM 占空比值（0~99），存储在 ROM 中的 `sine_table[100]` 数组中。

2. **双相中断**：Timer0 工作在 Mode 1（16 位定时器），每个采样周期分为 LOW 相和 HIGH 相。LOW 相期间 PWM 输出低电平，HIGH 相期间输出高电平，两相的定时器重装载值由当前占空比决定。

3. **频率调节**：输出频率由公式 `f = 1 / (N × T_sample)` 决定，其中 N=100 为采样点数，T_sample 为每个采样点的定时周期。通过修改定时器中断周期（查 `sample_ticks[20]` 表），实现 1~20Hz 的频率调节。

### 正弦查找表

```
duty[i] = round((sin(2π·i/100) + 1) / 2 × 99), i = 0, 1, ..., 99
```

| 索引范围 | 占空比范围 | 对应正弦位置 |
|---------|-----------|------------|
| 0~49 | 50→99→50 | 正半周（上升+下降） |
| 50~99 | 50→0→50 | 负半周（下降+上升） |

## 三、总体设计方案

采用 **"单片机控制 + 按键调频 + 数码管显示 + 定时器中断 + 查表法输出 SPWM"** 的整体架构。

### 系统工作流程

1. 单片机上电初始化：配置 Timer0（SPWM）+ Timer1（显示/节拍）、I/O 口、全局变量（初始频率 1Hz）
2. Timer0 中断按正弦查找表顺序输出 SPWM 脉冲（高优先级，保证波形精度）
3. Timer1 中断驱动数码管动态扫描（~2ms/位），同时置 `g_tick` 系统节拍
4. 主循环以 `g_tick` 节拍驱动非阻塞按键扫描（计数消抖 + 长按连按）
5. 按键修改 `g_frequency` 变量，ISR 自动使用新频率查表
6. 主循环空闲时进入 IDLE 省电模式，任何中断自动唤醒

## 四、主要器件与电路设计

### （一）核心器件选型

**ST89C52 单片机**：作为主控单元，负责 SPWM 波形生成、频率调节与显示控制。其 8 位架构、丰富的 I/O 口与定时器资源满足设计需求。工作频率 11.0592MHz，机器周期约 1.085μs。

**两位一体共阳七段码 LED**：用于实时显示当前调制波频率（01~20），共阳型设计简化驱动电路，通过动态扫描（每位 2ms）实现两位数字稳定显示。

**辅助器件**：按键模块（K1/K2 频率调节）、驱动三极管（可选，增强 SPWM 输出带载能力）、11.0592MHz 晶振与复位电路（保证单片机稳定运行）。

## 五、软件设计与实现

软件采用模块化架构，按功能拆分为 4 个独立模块，通过 `config.h` 统一配置。整体结构如下：

```
main.c                  主程序入口，全局频率变量，IDLE 省电主循环
├── config.h            板级配置：晶振、引脚映射、跨模块共享常量
├── spwm.c / spwm.h     SPWM 核心：q/r 分解 + Timer0 双相中断
├── display.c / display.h  Timer1 中断驱动数码管扫描 + 系统节拍
└── key.c / key.h        按键扫描、计数消抖、长按连按
```

### （一）系统主控制（main.c + config.h）

`config.h` 集中定义所有硬件相关常量：晶振频率（`FOSC = 11059200UL`）、频率范围（`FREQ_MIN=1`, `FREQ_MAX=20`）、引脚映射（SPWM 输出 P3.7、数码管段选 P1、位选 P2.0/P2.1、按键 P3.2/P3.3），以及全局变量 `g_frequency` 的声明。

`main.c` 负责系统初始化和主循环控制：

1. **初始化顺序**：先依次调用 `SPWM_Init()`、`Display_Init()`、`Key_Init()`，最后 `EA=1` 开全局中断
2. **主循环**：以 `g_tick`（~2ms 节拍）驱动非阻塞按键扫描，根据返回的 `KeyEvent` 修改 `g_frequency`
3. **IDLE 省电**：无节拍时执行 `PCON |= 0x01` 进入 IDLE 模式，任何中断自动唤醒

全局频率变量 `g_frequency` 声明为 `volatile unsigned char`，初始值 1Hz，由主循环修改，ISR 自动读取最新值。

### （二）SPWM 波形生成（spwm.c / spwm.h）

采用查表法：预先计算 100 个正弦采样点的占空比数据，存储在 ROM 中的 `sine_table[100]` 数组。Timer0 工作在 Mode 1（16 位定时器），通过双相中断机制输出 SPWM：

- **LOW 相（phase=0）**：P3.7 输出低电平，查 sine_table 获取 duty，定时器装载低电平持续时间
- **HIGH 相（phase=1）**：P3.7 输出高电平，定时器装载高电平持续时间，查表索引前进一位

各频率的定时器计数值采用 q/r 分解法预存在 `tick_q[20]` 和 `tick_r[20]` 数组中：

```
sample_ticks[f] = tick_q[f] × 100 + tick_r[f]
ticks = tick_q[f] × duty + (tick_r[f] × duty) / 100
```

ISR 根据 `g_frequency` 查表，仅需 16 位运算（避免 32 位乘除），在 8051 上约 60~75 机器周期完成。Timer0 设为高优先级（PT0=1），可抢占 Timer1，保证波形精度。ISR 中不停定时器，让其在计算期间继续计数，最后读取实际经过的 tick 数补偿 ISR 耗时。设置最小相位钳位 `MIN_PHASE_TICKS=80`，防止定时器重装载值过小导致 ISR 来不及执行。

### （三）数码管显示（display.c / display.h）

采用 Timer1 中断驱动动态扫描，替代主循环阻塞方式：
1. Timer1 配置为 Mode 2（8 位自动重装），TH1=0 → 溢出周期 256 × 1.085μs ≈ 278μs
2. 软件分频 `DISP_TIMER_DIV=7` → 显示扫描周期 ≈ 1.94ms ≈ 2ms
3. ISR 中自动交替显示十位和个位，每次切换前先熄灭（消鬼影）
4. 频率 1~9 时十位自动消隐（P1 输出 0xFF），10~20 正常显示
5. ISR 同时置 `g_tick = 1`，作为主循环的系统节拍

共阳七段码查找表 `seg_table[10]` 存储在 ROM（`code` 段），Timer1 使用独立寄存器组 `using 2`，避免与主循环和其他 ISR 的寄存器冲突。

### （四）按键输入（key.c / key.h）

主循环在每个 `g_tick` 节拍（~2ms）调用 `Key_Scan()`，检测 K1（P3.3 频率+）和 K2（P3.2 频率-）状态。按键扫描采用状态机实现，三态转换：

- **IDLE → HELD**：按键按下，计数消抖（10 × 2ms ≈ 20ms）确认后触发一次事件
- **HELD → REPEAT**：持续按住达到长按阈值（200 × 2ms ≈ 400ms）后触发
- **REPEAT → REPEAT**：每隔重复间隔（40 × 2ms ≈ 80ms）连续触发

按键松开立即复位到 IDLE。返回 `KeyEvent` 枚举（KEY_UP / KEY_DOWN / KEY_NONE），由主循环处理频率加减，频率范围限制在 1~20Hz。非阻塞设计，不影响 SPWM 波形输出和数码管显示。

### （五）关键技术参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 晶振频率 | 11.0592 MHz | 标准波特率晶振 |
| 机器周期 | 1.085 μs | 12时钟周期/机器周期 |
| 正弦采样点 | 100 点/周期 | 角度间隔 3.6° |
| PWM 分辨率 | 100 级 (0~99) | 占空比范围 |
| 频率范围 | 1~20 Hz | 按键 1Hz 步进 + 长按连按 |
| Timer0 中断频率 @10Hz | 2000 次/秒 | 100点 × 2相 × 10Hz |
| Timer1 扫描周期 | ~2ms | 7 × 278μs ≈ 1.94ms |
| Timer0 优先级 | 高 (PT0=1) | 可抢占 Timer1，保证波形精度 |
| CPU 占用 @10Hz | ~8.7% | ISR 开销极低，空闲时 IDLE |
