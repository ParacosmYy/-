# 基于 ST89C52 的 SPWM 波形发生器

## 项目简介

本项目基于 ST89C52 单片机，实现一个 **1Hz~20Hz 可调的 SPWM 波形发生器**。

当前版本的软件架构已经按模块化方式整理，核心特性如下：

- `P3.7` 输出 SPWM 波形
- `P3.3 / P3.2` 按键调节频率
- 两位共阳数码管显示当前频率
- **数码管刷新严格由主循环 `while (1)` 完成，不通过定时器中断刷新**

## 硬件连接

### 引脚分配

| 单片机引脚 | 功能 | 说明 |
|-----------|------|------|
| P1.0~P1.6 | 七段数码管 a~g 段 | 段选，低电平点亮 |
| P2.0 | 数码管十位位选 | 低电平选中十位 |
| P2.1 | 数码管个位位选 | 低电平选中个位 |
| P3.2 | 频率减按键 | 低电平有效 |
| P3.3 | 频率加按键 | 低电平有效 |
| P3.7 | SPWM 输出 | 接示波器或后级驱动 |

### 数码管说明

本项目使用两位一体共阳数码管：

- 十位接 `P2.0`
- 个位接 `P2.1`
- 段选共用 `P1`

显示规则如下：

- `1~9Hz`：十位灭，个位显示数值
- `10~20Hz`：十位和个位正常显示

## 软件架构

### 模块划分

```text
main.c            程序入口
app.c / app.h     应用层调度
spwm.c / spwm.h   SPWM 输出模块
display.c / display.h  数码管显示模块
key.c / key.h     按键扫描模块
freq.c / freq.h   频率管理模块
tick.c / tick.h   系统节拍模块
config.h          板级配置
```

### 设计约束

当前代码遵循以下关键设计约束：

1. `Timer0` 仅用于 SPWM 输出
2. `Timer1` 仅用于产生按键扫描节拍
3. **数码管扫描不在中断中执行**
4. 主循环持续调用 `display_task()` 刷新数码管

### 主循环逻辑

主循环入口位于 `main.c`：

```c
while (1) {
    app_task();
}
```

应用层任务位于 `app.c`：

- 每次进入 `app_task()` 都调用 `display_task()`
- `display_task()` 每次只刷新一位数码管
- `Timer1_ISR()` 只调用 `tick_notify()`，不参与数码管刷新

因此，当前实现中数码管刷新路径为：

```text
main -> app_task() -> display_task()
```

这条路径完全运行在前台 `while` 循环中。

## 当前实现说明

### 数码管模块

数码管显示模块位于 `display.c / display.h`，职责如下：

- 保存待显示数值
- 拆分十位和个位
- 在前台轮询中完成动态扫描

### 应用层模块

应用层位于 `app.c / app.h`，职责如下：

- 初始化显示、按键、节拍
- 驱动主循环任务
- 处理按键事件并更新频率显示

### SPWM 模块

SPWM 模块位于 `spwm.c / spwm.h`，职责如下：

- 使用查表法生成正弦包络
- 使用 `Timer0` 双相中断输出 SPWM
- 不处理显示和按键逻辑

## 注意事项

1. 如果数码管左右位和实际硬件相反，请检查 `DIG_TENS` 与 `DIG_ONES` 的接线顺序
2. 如果显示存在重影，请优先检查数码管公共端驱动能力和硬件接线
3. 当前软件设计要求数码管刷新只能在主循环中执行，不应重新改回中断扫描
